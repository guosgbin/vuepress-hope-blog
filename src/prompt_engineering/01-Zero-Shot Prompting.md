## 简介

**Zero-Shot Prompting**：零样本提示法，不给模型任何示例，只靠一句指令，让模型直接完成任务。

一句话总结：**不用教、不用举例，直接让 AI 干活。**

和其他的提示方式的区别：

1. **Zero-Shot（零样本）**

   不给例子，直接下指令。

2. **Few-Shot（少样本）**

   给 1～3 个例子，再让 AI 照着做。

## 工作原理

大模型在训练时已经学过海量知识，包括：

- 语言理解
- 分类
- 推理
- 翻译
- 总结
- 代码

所以**不用再教它**，只要说清楚任务，它就能直接做。

> 当**零样本提示（zero-shot）不起作用**时，建议在提示词中提供**示范或示例**，这就形成了**少样本提示（few-shot prompting）**。

## 案例

**案例 1：情感判断**

*Prompt*：

```
判断下面这句话的情绪是积极、消极还是中性。
今天心情特别好。
```

*模型输出*：

```
这句话的情绪是积极
```

**案例 2：翻译**

*Prompt*：

```
把下面句子翻译成英文：
我正在学习 AI 提示词工程。
```

*模型输出*：

```
I am learning AI prompt engineering.
```

**案例 3：文本分类**

*Prompt*：

```
把下面文本分类为：科技、娱乐、体育，只输出类别。
国足在比赛中以 2 比 1 战胜对手。
```

*模型输出*：

```
体育
```

**案例 4：逻辑推理**

*Prompt*：

```
请推理：
所有的猫都有尾巴。
咪咪是一只猫。
请问咪咪有尾巴吗？
```

*模型输出*：

```
根据您提供的前提进行逻辑推理：
大前提：所有的猫都有尾巴。
小前提：咪咪是一只猫。
结论：既然咪咪属于“猫”这个群体，而该群体的所有成员都具备“有尾巴”的特征，那么咪咪必然也有尾巴。
答案：是的，咪咪有尾巴。
```

**案例 5: 代码生成**

*Prompt*：

```
写一个 go 函数，计算两个数的和。
```

*模型输出*：

```go
package main

import "fmt"

// Add 计算两个整数的和
func Add(a, b int) int {
	return a + b
}

func main() {
	result := Add(3, 5)
	fmt.Println("3 + 5 =", result) // 输出: 3 + 5 = 8
}
```
